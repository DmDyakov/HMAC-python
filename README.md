# HMAC Message Signing and Verification Service

REST API сервис для подписи и проверки целостности сообщений по алгоритму HMAC-SHA256.

## Ограничения учебной реализации

Данный проект является учебным и имеет следующие ограничения:

1. **HMAC ≠ асимметричная ЭП**: Используется симметричный алгоритм HMAC-SHA256, который не обеспечивает неотказуемость. Обе стороны (отправитель и получатель) используют один и тот же секретный ключ.
2. **Нет многоключевой валидации**: Система работает с одним секретным ключом для всех операций.
3. **Простая ротация**: Утилита ротации просто заменяет ключ в конфигурации, без механизмов версионирования или плавного перехода.
4. **Нет шифрования**: Сервис только подписывает и проверяет сообщения, но не шифрует их содержимое.
5. **Требуется безопасный обмен ключом**: Секретный ключ должен быть передан между сторонами заранее по защищенному каналу.

## Требования к окружению и установка

Для работы потребуется менеджер зависимостей `uv`. Установите его по инструкции из GitHub репозитория astral-sh/uv:
https://github.com/astral-sh/uv?tab=readme-ov-file#installation

Установка зависимостей:
```bash
make sync
```

## Конфигурация

Настройки приложения хранятся в файле `config.json`:

```json
{
  "hmac_alg": "SHA256",
  "secret": "<base64-encoded-secret>",
  "log_level": "info",
  "listen": "127.0.0.1:8080",
  "max_msg_size_bytes": 1048576
}
```

### Формат конфигурации:

- **hmac_alg**: Алгоритм HMAC (поддерживается только SHA256)
- **secret**: Секретный ключ в кодировке base64 (без паддинга)
- **log_level**: Уровень логирования (debug, info, warning, error, critical)
- **listen**: Адрес и порт для запуска сервера (формат: "host:port")
- **max_msg_size_bytes**: Максимальный размер сообщения в байтах

### Генерация секретного ключа

Для генерации нового секретного ключа можно использовать Python:

```bash
python -c "import base64, secrets; print(base64.urlsafe_b64encode(secrets.token_bytes(32)).rstrip(b'=').decode())"
```

Или воспользоваться утилитой ротации (см. ниже).

## Запуск сервера

Поднять сервер с API можно через команду Make:

```bash
make run/api
```

Сервер будет доступен по адресу, указанному в конфигурации (по умолчанию http://127.0.0.1:8080).

## API Endpoints

### POST /sign

Подписывает сообщение с помощью HMAC-SHA256.

**Запрос:**
```json
{
  "msg": "string"
}
```

**Ответ (200):**
```json
{
  "signature": "base64url-encoded-signature"
}
```

**Ошибки:**
- 400: Пустое или невалидное сообщение
- 413: Превышен максимальный размер сообщения
- 422: Неверный формат запроса
- 500: Внутренняя ошибка сервера

### POST /verify

Проверяет соответствие подписи сообщению.

**Запрос:**
```json
{
  "msg": "string",
  "signature": "base64url-encoded-signature"
}
```

**Ответ (200):**
```json
{
  "ok": true
}
```
или
```json
{
  "ok": false
}
```

**Ошибки:**
- 400: Пустое сообщение или невалидный формат подписи
- 413: Превышен максимальный размер данных
- 422: Неверный формат запроса
- 500: Внутренняя ошибка сервера

## Примеры использования с curl

### Подпись сообщения:

```bash
curl -sS -X POST http://localhost:8080/sign \
  -H 'Content-Type: application/json' \
  -d '{"msg":"hello"}'
```

Ответ:
```json
{"signature":"кXRzZWNyZXQ="}
```

### Проверка подписи:

```bash
curl -sS -X POST http://localhost:8080/verify \
  -H 'Content-Type: application/json' \
  -d '{"msg":"hello","signature":"скопированная_подпись"}'
```

Ответ (подпись верна):
```json
{"ok":true}
```

### Проверка с неверной подписью:

```bash
curl -sS -X POST http://localhost:8080/verify \
  -H 'Content-Type: application/json' \
  -d '{"msg":"hello","signature":"invalid_signature"}'
```

Ответ (подпись неверна или повреждена):
```json
{"ok":false}
```
(или ошибка 400, если формат подписи не base64url)

## Ротация секретного ключа

Для смены секретного ключа используйте утилиту ротации:

```bash
python rotate_secret.py
```

Утилита сгенерирует новый случайный ключ, заменит его в `config.json` и выведет старый и новый ключи.

**Важно**: После ротации ключа все ранее созданные подписи станут недействительными!

## Инструкция по запуску

1. **Клонируйте репозиторий и установите зависимости:**
```bash
git clone <your-forked-repo>
cd hmac-python
make sync
```

2. **Настройте конфигурацию:**
```bash
# Создайте config.json из примера
cp example.config.json config.json

# Или сгенерируйте новый секретный ключ
python -c "import base64, secrets; print(base64.urlsafe_b64encode(secrets.token_bytes(32)).rstrip(b'=').decode())"
# Затем вставьте полученный ключ в config.json
```

3. **Запустите тесты:**
```bash
make test
```

4. **Запустите сервер:**
```bash
make run/api
```

5. **Проверьте работу:**
```bash
# Подписать сообщение
curl -X POST http://localhost:8080/sign \
  -H 'Content-Type: application/json' \
  -d '{"msg":"hello"}'

# Проверить подпись (используйте signature из предыдущего ответа)
curl -X POST http://localhost:8080/verify \
  -H 'Content-Type: application/json' \
  -d '{"msg":"hello","signature":"полученная_подпись"}'
```

6. **Ротация ключа (опционально):**
```bash
python rotate_secret.py
```
